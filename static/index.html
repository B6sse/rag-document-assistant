What it can do

Upload a PDF and start a conversation with your documents.

💬

Conversation history

Every chat is saved automatically. Pick up where you left off or start a new conversation at any time.

🔍

Semantic search

Finds relevant passages even when your wording differs from the source text. Embeddings run fully locally so no extra API key is needed.

📋

Search or summarize

Switch between focused search for specific facts and deep summarization that reads the full document before answering.

📍

Source references

Every answer cites the document and page number so you can verify the information yourself.

🗂️

Documents per chat

Each conversation has its own set of documents. Files uploaded in one chat never interfere with another.

🔒

Runs locally

The vector database and embedding model run on your own machine. Only the final question and retrieved text are sent to Claude.

How it works

From PDF to answer in four steps.

01

Upload a PDF

Click the plus button next to the input field and select a PDF. The file is tied to the current conversation so it stays separate from your other chats.

02

The document is indexed

The text is split into overlapping chunks and each chunk is converted to a vector using a local embedding model. The vectors are stored in ChromaDB on your machine.

03

Your question is matched

When you ask something, the same embedding model converts your question to a vector and retrieves the most relevant chunks from the database. In Summarize mode a larger portion of the document is retrieved to give a fuller picture.

04

Claude writes the answer

The retrieved passages are sent to Claude together with your question. Claude reads the context and returns a precise answer with page references, streamed back to you in real time.
